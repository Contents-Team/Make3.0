# 업데이트 워크플로우 — 팀원 ↔ Claude 협업 규약

> 이 레포는 두 종류의 변경 로그를 **분리** 관리한다. 팀원이 업데이트할 게 생기면 Claude(AI 에이전트)에게
> 아래 형식으로 요청하고, Claude는 문서 수정 → 해당 로그 기록 → PR 생성까지 수행한다. **머지(승인)는 사람이 한다.**

## 0. 두 로그 구분 (어디에 기록되나)

| 변경 종류 | 기록 위치 | 예 |
|---|---|---|
| **문서·네이밍·포맷 규칙** 변경 | [`CHANGELOG.md`](../CHANGELOG.md) (SemVer, v1→v7 …) | `.make` 포맷 명세 보강, 네이밍 규칙 개정 |
| **Make 제품(설치파일/빌드)** 기능 변경 | [`PRODUCT_CHANGELOG.md`](../PRODUCT_CHANGELOG.md) (버전+분석일) | 새 설치파일에서 컴포넌트/셰이더/기능 추가·수정 |

헷갈리면 기준 하나 — **"문서를 고치는 일"이면 CHANGELOG.md, "제품 빌드가 바뀐 걸 기록"이면 PRODUCT_CHANGELOG.md.**

## 1. 팀원이 Claude에게 명령하는 법 (상황별)

### A. 네이밍/포맷 규칙을 바꾸고 싶을 때
> 예: "MAKE_FORMAT 네이밍 규칙에 모달 팝업용 상태 어휘 추가하고 싶어. `Expanded`/`Collapsed` 넣어줘."

Claude가 하는 일: `doc/MAKE_FORMAT.md` 수정 → `CHANGELOG.md` `Unreleased`에 항목 추가 → 브랜치+PR 생성.
요청 시 **근거/의도**를 한 줄 덧붙이면 좋다("이유: VR 모달이 접힘 상태를 구분해야 함").

### B. 새 설치파일(빌드)이 나왔을 때  ← 가장 흔한 케이스
> 예: "`MV 3.0`에 `MakeInstaller_3.1.0.9.exe` 새로 받아놨어. 직전 버전이랑 비교해서 뭐가 바뀌었는지 확인해줘."

Claude가 하는 일:
1. 설치파일을 읽기 위치로 추출(7-Zip, 실행 아님) — 파일이 이미 추출돼 있으면 그 폴더 알려주면 됨.
2. 빌드 지문화(파일 해시·셰이더·씬·에셋 + C# 어셈블리 공개 타입/메서드, **실행 없이**).
3. 직전 버전 지문과 diff → `PRODUCT_CHANGELOG.md`에 `[Added]/[Modified]/[Removed]` 항목(버전·분석일) 추가.
4. 포맷/컴포넌트에 영향 있으면 `doc/MAKE_FORMAT.md`도 보강(그 변경은 `CHANGELOG.md`에도 기록).
5. 브랜치+PR 생성.

**필요한 정보만 주면 된다**: 설치파일 이름(또는 추출 폴더 경로), 직전 비교 대상(모르면 Claude가 마지막 기록 버전을 씀).

### C. Unreleased를 정식 릴리스로 끊고 싶을 때
> 예: "지금 CHANGELOG Unreleased 내용을 `0.1.7`로 릴리스 처리해줘. 날짜는 오늘로."

Claude가 하는 일: `Unreleased` 항목을 `## [0.1.7] - <오늘>`로 이동, 새 빈 `Unreleased` 생성 → PR.

## 2. 명령할 때 지켜주면 좋은 것

- **한 번에 한 책임**: "README도 고치고 네이밍도 바꾸고 새 빌드도 분석해줘"처럼 섞지 말 것 → PR이 뒤엉킨다. 건별로.
- **날짜는 절대값**: "어제"가 아니라 "2026-07-01"처럼. (Claude는 실행 시점 실제 날짜를 씀)
- **원본 파일 위치 명시**: 새 빌드는 어디에 뒀는지(경로/파일명) 알려주기.
- **손대면 안 되는 것 상기**: `LICENSE`·`SECURITY.md`·기존 `doc/adr/*` 본문은 수정 금지(AGENTS.md). 규칙 변경은 supersede.

## 3. PR / 승인 흐름 (사람이 머지)

1. Claude가 브랜치 생성 → 변경 커밋 → **PR 생성**(제목·본문·`Model Used` 채움, `CHANGELOG`/`PRODUCT_CHANGELOG` 갱신 포함).
2. 사람(메인테이너)이 PR 리뷰 → 승인 → **머지**. Claude는 머지하지 않는다.
3. CI가 있으면 green 확인 후 머지.
4. 상세 규약·DoD·금지사항은 [`AGENTS.md`](../AGENTS.md), 기여 절차는 [`CONTRIBUTING.md`](../CONTRIBUTING.md).

## 4. 한 줄 요약

> **"무엇이 바뀌었는지 + 관련 파일/버전"만 Claude에게 말하면, Claude가 올바른 로그에 기록하고 PR까지 만든다.
> 사람은 리뷰·승인·머지만 한다.**
